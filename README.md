# TikTok Trend Bot

Персональний трендовий дайджест TikTok у Telegram + AI-чат.
Пайплайн: **Apify** (`clockworks/tiktok-scraper`) → **Claude** (фільтр + розбір) → **Telegram**.

## Можливості

- 🚀 Онбординг: новий юзер бачить банер-опис бота ("🚀 Почати" / "📋 Що вміє бот"),
  а не одразу меню дій; юзер, який вже обрав нішу раніше — /start одразу відкриває
  звичне меню
- 📊 Дайджест топ-5 трендових відео по ніші з поясненням «чому віральне» і «що вкрасти»
- 🔥 Динамічні підтеги — після дайджесту/музики бот аналізує ВСІ хештеги відео
  в пулі (не лише ті, за якими шукали) і пропонує топ-5 таких, що спливають
  (частота × середня velocity), з кнопками додати їх до пошуку (Pro)
- ▶️ Кнопка «Далі» — наступні 5 відео зі СПІЛЬНОГО кеш-пулу без повторів (новий
  Apify-запуск тільки коли пул вичерпано/старіший за `POOL_TTL_HOURS` — і один раз
  на всіх юзерів з однаковою нішею+регіоном, а не на кожного окремо)
- 🌍 Регіон трендів: 8 преset-країн кнопками + довільний ISO-код (Pro). Якщо
  country-proxy недоступний на Apify-плані — бот автоматично відкочується на
  Global і попереджає
- 🏷 Ніша + мульти-вибір preset-хештегів чекбоксами + власні хештеги через кому (Pro)
- 🎵 «Музика в тренді» — топ звуків (5 Pro / 3 Free) за частотою `musicId` у пулі +
  фільтр стилю через Claude (за назвою/автором треку, не за аудіо-аналізом)
- 💬 Режим запитань до Claude з контекстом трендів твоєї ніші
- 🌐 Інтерфейс на 6 мовах (укр, англ, порт, ісп, фр, рос) — вибір при першому /start,
  зміна через `/language`
- 💎 Тарифи Free/Pro: Free — 1 ніша, тільки Global, 1 дайджест/добу без «Далі»,
  тільки preset-хештеги, топ-3 музики; Pro — без обмежень. Оплата поки ручна
  (адмін через `/mark_paid`), Stripe/Telegram Payments не підключені
- 🏠 Постійна кнопка «Меню», fallback-меню замість тиші на незрозумілі повідомлення
- 🗄 Стан — у SQLite, переживає рестарти Railway

## Команди

Юзер: `/start` `/niche` `/digest` `/ask` `/settings` `/cancel` `/language` `/upgrade`

Адмін (тільки `ADMIN_CHAT_ID`): `/set_tier <chat_id> <free|pro>`,
`/mark_paid <chat_id> <days> [amount]`, `/users`, `/stats`, `/revenue`

Щодня о `DIGEST_HOUR` (UTC) бот шле дайджест автоматично в `CHAT_ID`.
Щодня о 00:05 UTC — перевірка прострочених Pro-підписок (даунгрейд на Free).

## Змінні середовища

| Змінна | Обов'язкова | Дефолт | Опис |
|---|---|---|---|
| `TELEGRAM_TOKEN` | так | — | токен бота від @BotFather |
| `ANTHROPIC_API_KEY` | так | — | ключ Anthropic API |
| `APIFY_TOKEN` | так | — | токен Apify |
| `CHAT_ID` | ні | — | chat_id для щоденного автодайджесту |
| `HASHTAGS` | ні | `football,beauty` | дефолтні хештеги, якщо ніша не вибрана |
| `RESULTS_PER_TAG` | ні | `20` | скільки відео тягнути на хештег |
| `DIGEST_HOUR` | ні | `8` | година автодайджесту (UTC) |
| `DB_PATH` | ні | `bot.db` | шлях до SQLite-бази |
| `POOL_SIZE` | ні | `30` | скільки відео тримати в кеш-пулі |
| `POOL_TTL_HOURS` | ні | `3` | через скільки годин пул вважати застарілим |
| `ADMIN_CHAT_ID` | ні | — | chat_id, якому доступні адмінські команди (напр. `734504128`) |
| `ADMIN_USERNAME` | ні | `admin` | @username адміна без "@", показується в `/upgrade` (напр. `kholstynin`) |

⚠️ Значення для цих (і будь-яких інших) змінних задаються ТІЛЬКИ в Railway →
Variables, ніколи в коді. Правити `main.py`, підставляючи туди свій chat_id
чи username замість назви змінної — типова помилка, яка ламає перевірку
адміна (код почне шукати неіснучу env-змінну з назвою на кшталт `734504128`
і завжди отримувати `None`). Адмін, вказаний у `ADMIN_CHAT_ID`, автоматично
отримує тариф Pro при кожному старті бота (`post_init`), без потреби
виконувати `/set_tier` вручну.

## Деплой на Railway

⚠️ **Обов'язково для персистентності:** Railway стирає файлову систему при кожному
redeploy. Щоб SQLite-база (ніші, показані відео, кеш) переживала деплої:

1. Підключи **Volume** до сервісу (наприклад, з точкою монтування `/data`).
2. Постав змінну `DB_PATH=/data/bot.db` (шлях має бути всередині точки монтування).

Без Volume база живе лише до наступного деплою.

Регіональні тренди використовують `proxyConfiguration.apifyProxyCountryCode` —
для них на Apify-плані має бути доступ до residential/country proxy. Якщо доступу
немає, бот сам відкотиться на Global.

## Відомі обмеження (свідомі трейд-оффи)

- **Спільний пул і власні хештеги Pro:** `trend_pool` кешується на
  `(niche_key, region)` без прив'язки до юзера — це навмисно (мета: один
  Apify-запуск на всіх, а не на кожного). Free-юзери не можуть змінювати
  хештеги, тому для них це завжди коректно. Pro-юзер із власними хештегами
  може тимчасово побачити пул, підтягнутий під чиїсь ІНШІ хештеги тієї ж
  ніші, поки кеш не оновиться (до `POOL_TTL_HOURS`). Якщо це стане проблемою —
  варто ключувати пул по хештегах, а не по `niche_key`.
- **Адмінські команди** (`/set_tier`, `/mark_paid`, `/users`, `/stats`,
  `/revenue`) не перекладені — це інструмент для одного оператора, не
  частина юзерського UX.
- **`/stats` "Дайджестів сьогодні"** — сума `digest_count` за день (не
  кількість рядків), щоб коректно рахувати кілька дайджестів на добу для Pro.
- **Переклади** (uk/en/pt/es/fr/ru) написані вручну для короткого UI-тексту;
  варто дати носіям мови звірити перед широким запуском.
- **Оплата ручна:** Stripe/Telegram Payments не підключені — тариф Pro
  ставить лише адмін через `/mark_paid` або `/set_tier`.
- **Поле хештегів у Apify-відповіді:** `clockworks/tiktok-scraper` нормалізує
  хештеги відео в `"hashtags": [{"id","name","title","cover"}]`. Це підтверджено
  зовнішнім пошуком (офіційна документація Apify недоступна для автоматичного
  фетчу — 403), тому `item_hashtag_names()` захисно читає `name` або `title`,
  приймає й масив рядків, і має легасі-фолбек на `textExtra[].hashtagName` на
  випадок відхилення схеми в майбутньому. Варто звірити на реальному прогоні.
